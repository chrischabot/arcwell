# Project Control

Rules:

- Use projects for named work streams, aliases, summaries, and status.
- Resolve natural references before acting.
- If a reference is ambiguous, present candidates instead of guessing.
- Preserve follow-up context when a channel asks about "the other one" or "that project."
- Keep project records factual and concise.

Useful tools:

- `project_create`
- `project_resolve`
- `project_list`
- `project_status_get`
- `project_status_record`
- `project_status_sync_record`
- `channel_list`
- `ops_snapshot`
